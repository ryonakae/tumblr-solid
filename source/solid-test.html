Webサイトのアニメーションについて

2014.02.18 / 1 week ago

なんでこれやりたいか

  • ファイルサイズが大きすぎるのでストレージ圧迫する
  • ファイルサイズが大きすぎるのでロードなどにも時間がかかる
  • その結果写真の選別に時間がかかる
  • 現像時のPCへの負荷が尋常じゃない
  • 書き出し時のPCへの負荷が尋常じゃない
  • 不要な写真(失敗とか)も全部RAWで置いておいておくのに意味あるのかと悩む
  • かと言って必要な写真だけRAWで残して後は消すorJPGだけ残す、というのも面倒

ああああああ

普通に書くのとそんなに労力変わらんじゃねーかという感じだけど、mixin使うとconfig.rbにて設定したfontディレクトリを参照してくれる。font/hoge.eotではなくhoge.eotって書いとけばいいということ。

@font-face {
  font-family: "Hoge";
  src: url('font/hoge.eot');
  src: url('font/hoge.eot') format('eot'),
       url('font/hoge.eot') format('embedded-opentype'),
       url('font/hoge.woff') format('woff'),
       url('font/hoge.ttf') format('truetype'),
       url('font/hoge.svg') format('svg');
  font-weight: "normal";
}

雪降ってた

昨日、もう一昨日か、京都でクッソ雪降ってたのでテンション上がってた。先日も降ってたけど、夜に降りまくって朝になったら積もってるだけだったので、リアルタイムに雪降ってる中歩けて大変良かった。何回か滑って転びそうになったけど。朝起きて、カーテン開けて外見た瞬間にウッキウキになった。

家の前の道とか結構雪積もってて、僕の家は山の中腹辺りにあるので坂道が多く、これは自転車無理だわ、と歩いて出勤。普段は持ち歩かないGRを持って、歩きながらちょいちょい写真撮った。

この写真、わりと気に入ってる。ヤマトのお兄さんが通りかかって、「雪なのに大変だなー」と思っていたら後ろから郵便局のお兄さんがバイクで来て、「雪なのに大変だな―…ってっさっきも思ったぞ!?」という感じで咄嗟に振り向いて撮った1枚。GRのスナップシューターっぷりが遺憾なく発揮された写真である。流石にトリミングはしたけど。

これは最寄り駅で電車待ってる時に撮った写真、こんな感じで大変風情があった。Lightroomで歪み補正かけたら下のレールが気持ち悪いことになってしまったけど、気にしない…。
本当にこれくらい雪降ってて超良かった。去年も雪が降った記憶があるけど、こんな大粒の雪ではなくもっと小ぶりで、おかげで写真撮っても埃が写っているようにしか見えなかった。これだけ大きな雪だと写真でもはっきり分かる。

なんでこれやりたいか

雪の日は空も地面も全体的に白くなるので、カメラが「おっ、なんか白いな、今めっちゃ明るいじゃん！」と勘違いする。なのでシャッター速度が上がり、雪とかはこんな感じで止まって写る感じになる。デメリットはシャッター速度が上がる分、光を取り込む時間が短くなるので、絞りを開けないといけない。絞ると真っ暗な写真が出来る(最近のカメラはそんなに馬鹿ではないが)。あと、カメラってあまりにも白い箇所は暗く写そうとするから、写真が全体的に暗くなる。なので後で補正するとか、露出補正とかで明るくしないとダメ。

しかしこのGRの色設定は良い感じだと我ながら思う。コントラスト高すぎるけど、これくらいドギツい方がぱっと見目を引くから僕は好きだ。ホワイトバランスは暖色より寒色の方が僕は好み。朝っぽいからかもしれない。朝って光が青緑っぽく見えるし、僕は凄く好き。でも朝が弱すぎて朝の光をあんまり見られないから、そこに憧れがあるのかもしれない(適当)

東京の方は生活に支障きたすレベルで雪降ってるっぽいけど、ぶっちゃけうらやましい。写真撮り放題じゃん！と。京都や大阪はこの日の昼には雨に変わってしまったので残念だった。次のチャンスは来年のこの時期かな。
